# Git 工作流常用命令速查表

本文档总结了日常开发中最常用的 Git 命令及其具体作用，并附带了面试中的加分话术建议。

## 1. 基础初始化
| 常用命令 | 具体作用 | HR/技术面试加分点 |
| :--- | :--- | :--- |
| `git init` | 在当前目录创建一个新的 Git 仓库。 | 提到“项目初始化时的结构设计”。 |
| `git clone [url]` | 从远程服务器（如 GitHub）下载完整项目。 | 提到“克隆特定分支以节省带宽”。 |

## 2. 本地开发循环
| 常用命令 | 具体作用 | HR/技术面试加分点 |
| :--- | :--- | :--- |
| `git status` | 查看当前文件的修改状态（暂存/未暂存）。 | 展示养成“随时查看状态”的严谨习惯。 |
| `git add .` | 将所有修改过的文件添加到暂存区 (Staging)。 | 提到“只添加必要文件，避免污染仓库”。 |
| `git commit -m "msg"` | 将暂存区内容提交到本地仓库，生成记录。 | 强调 **“规范化提交信息”** 的重要性。 |

## 3. 分支管理
| 常用命令 | 具体作用 | HR/技术面试加分点 |
| :--- | :--- | :--- |
| `git branch [name]` | 创建一个新功能分支。 | 提到“功能开发必须在独立分支进行”。 |
| `git checkout [name]` | 切换到指定分支（现代常用 `git switch`）。 | 切换前确保当前工作区是干净的。 |
| `git merge [name]` | 将指定分支的代码合并到当前分支。 | 提到“处理冲突时的耐心与细致”。 |
| `git rebase` | 变基操作，让提交历史变成一条直线。 | 进阶：说明你追求“整洁的历史记录”。 |

## 4. 远程协作
| 常用命令 | 具体作用 | HR/技术面试加分点 |
| :--- | :--- | :--- |
| `git pull` | 拉取远程代码并直接合并到本地。 | 提到“开发前先 Pull，减少冲突概率”。 |
| `git fetch` | 只下载远程更新，不自动合并（更安全）。 | 提到“先对比差异再决定是否合并”。 |
| `git push` | 将本地提交推送到远程仓库。 | **严禁**使用 `-f` (强制推送) 覆盖队友代码。 |

## 5. 调试与回滚
| 常用命令 | 具体作用 | HR/技术面试加分点 |
| :--- | :--- | :--- |
| `git log --oneline` | 查看简洁的提交历史记录。 | 快速定位 Bug 产生的版本节点。 |
| `git reset --hard [ID]` | 强制回退到某个历史版本。 | 谨慎使用，通常用于本地环境灾难恢复。 |
